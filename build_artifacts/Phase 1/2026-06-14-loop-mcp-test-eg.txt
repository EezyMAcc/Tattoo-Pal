‚Źļ The isolation boundary, explained plainly. The key flag is -v 
  "$PWD":/workspace:

  - A volume mount (-v) glues a folder on your Mac to a path inside the
  container. Here, your tattoo-feed folder ‚ÜĒ /workspace. It's one shared folder,
  not a copy ‚ÄĒ a file written on either side appears on the other instantly.
  - Because it's the only mount, the container's view of your Mac is exactly 
  this one folder. It cannot see ~/Documents, your other projects, SSH keys, or
  system files. They simply don't exist inside the container.
  - Everything else in the container (the OS, Python, Node) is the image's own
  throwaway filesystem. With --rm, the container is deleted the moment it exits
  ‚ÄĒ that scratch filesystem vanishes, but /workspace (your real folder) persists
  because it lives on your Mac.

  The other flags: -it = interactive terminal (a usable shell); -w /workspace =
  start in that folder.

  How you'll authenticate Claude Code (first run inside the container)

  The image ships the claude CLI but no credentials ‚ÄĒ that's correct, you never
  bake secrets into an image. You log in inside a running container:

  1. Start a shell: ./run-loop.sh
  2. At the container prompt, run claude. On first launch it triggers a
  browser-based OAuth login: it prints a URL, you open it on your Mac, approve,
  and paste the code back. (Alternatively, if you have an Anthropic API key, you
  can set ANTHROPIC_API_KEY ‚ÄĒ but the interactive login is the normal path.)

  ‚ö†ÔłŹ One caveat to be aware of: that login is stored in the container's home
  directory (/root/.claude), which is part of the throwaway filesystem. Because
  run-loop.sh uses --rm, when you exit the container the login is discarded and
  you'd have to log in again next time. That's fine for a single overnight run
  (log in, then kick off the build in the same session). If you later want the
  login to persist across container restarts, we'd add a second mount for the
  credentials directory ‚ÄĒ tell me when you're ready and I'll wire it in.

  How the built code ends up back in this folder

  This is the payoff of the volume mount. Inside the container, Claude Code does
  all its work in /workspace. But /workspace is your tattoo-feed folder on the
  Mac ‚ÄĒ the same bytes, not a copy. So:

  - When Claude writes src/tattoo_feed/..., commits to the feat/auto-build
  branch, or creates BLOCKERS.md, those files appear immediately in this folder
  on your Mac.
  - When the container exits and is deleted, none of that work is lost ‚ÄĒ it was
  never inside the container's disposable layer; it was always on your host via
  the mount.

  In short: the container is disposable, your code is permanent, and the mount
  is the one bridge between them.

      126 +## Design decisions                                           
      127 +                                                              
      128 +- **Two-layer split (core / server).** MCP concepts never leak
          + into `core`;                                                 
      129 +  business logic never leaks into `server`. This is what makes
          + a phase-2 GUI a                                              
      130 +  bolt-on rather than a rewrite.                              
      131 +- **JSON-file persistence behind a `Repository` interface.** S
          +imple, inspectable,                                           
      132 +  and swappable. Writes are **atomic** (temp file + `os.replac
          +e`) so a crash                                                
      133 +  mid-write can never corrupt a store.                        
      134 +- **Pydantic v2 frozen models** for everything crossing a boun
          +dary, so external                                             
      135 +  data is validated once and treated as immutable values there
          +after.                                                        
      136 +- **Typed error hierarchy** (`TattooFeedError` and friends). E
          +very external                                                 
      137 +  failure maps to a typed error; nothing raises bare exception
          +s across a                                                    
      138 +  boundary, so the client always gets a readable message inste
          +ad of a stack                                                 
      139 +  trace.                                                      
      140 +- **Lazy credentials.** The server boots and lists its tools w
          +ith no network and                                            
      141 +  no real credentials; the token is only read when a tool actu
          +ally calls                                                    
      142 +  Instagram.                                                  
      143 +- **Hermetic tests.** All Instagram HTTP is mocked with `respx
          +`; there are zero                                             
      144 +  live network calls in the test suite. (`mypy --strict`, `ruf
          +f`, and a 90%                                                 
      145 +  coverage floor are enforced.)                               
      146 +- **Images only where they earn their context.** Only `next_in
          +spiration` returns                                            
      147 +  a rendered image ‚ÄĒ the one-at-a-time conversational moment. 
          +`get_feed` stays                                              
      148 +  metadata-only to keep the context window light.             
      149 +                                                              
      150 +---                                                           
      151 +                                                              
      152 +## Limitations (by design)                                    
      153 +                                                              
      154 +- **No video.** Video posts are filtered out entirely at the G
          +raph-client layer                                             
      155 +  and never enter the feed, inspiration, or stores.           
      156 +- **Carousels show the first image only.** Multi-image expansi
          +on is out of scope.                                           
      157 +- **Manual token refresh.** There is no automatic token refres
          +h. When the token                                             
      158 +  expires, tools fail with a clear `TokenExpiredError` telling
          + you to mint a new                                            
      159 +  long-lived token and update `IG_ACCESS_TOKEN`.              
      160 +- **Preview sizing is fixed.** Previews are capped at **640px 
          +on the long edge**,                                           
      161 +  aspect ratio preserved, never upscaled, re-encoded as **JPEG
          + quality 85**.                                                
      162 +- **`record_preference` is propose-then-confirm.** The tool pe
          +rsists whatever it                                            
      163 +  is given; the *discipline* of proposing the observation to y
          +ou and getting your                                           
      164 +  explicit confirmation **before** the tool is called lives in
          + the tool's                                                   
      165 +  description, so the calling assistant honours it.           
      166 +- **No write access to Instagram.** No posting, commenting, or
          + messaging ‚ÄĒ this                                             
      167 +  is strictly read-and-curate.
